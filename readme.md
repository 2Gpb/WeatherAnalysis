# Weather analysis app

В ходе выполнения задания я попробовал использовать параллельную обработку данных с помощью multiprocessing.Pool. Для этого я разделил исторические данные по городам и выполнял расчеты независимо для каждого города в отдельном процессе.

Но в моем случае использование параллельности оказалось менее эффективным, чем обычная последовательная обработка с помощью pandas. Это связано с тем, что объем данных для одного города относительно небольшой, а накладные расходы на создание процессов, передачу данных между ними и последующее объединение результатов оказываются больше, чем выигрыш от параллельного выполнения. В результате последовательный вариант работает быстрее и выглядит более оправданным для данной задачи.

Также я попробовал использовать синхронный и асинхронный подходы для получения текущей погоды, используя библиотеки requests и aiohttp. Существенной разницы во времени выполнения между этими вариантами я не заметил. Это объясняется тем, что в приложении выполняется всего один небольшой get запрос, и использование асинхронности не дает ощутимого преимущества.

В связи с этим для данного приложения я считаю более логичным использовать синхронный подход с библиотекой requests, так как он проще в реализации и полностью покрывает текущие требования. Асинхронный подход может быть полезен в случае, если потребуется выполнять несколько запросов одновременно или работать с большим количеством городов.
